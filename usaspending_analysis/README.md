# Federal IT Contract Longevity: Disproving the "Mega-Vendor" Incumbency Myth

## Project Overview
This repository contains the full automated data engineering pipeline, machine learning predictive architecture, and scholarly manuscript for the original research paper: *"The Longevity of Public Sector IT: Disproving the 'Mega-Vendor' Incumbency Myth Utilizing Dynamic Random Forest Diagnostics on USAspending Data."*

The U.S. federal government allocates over $100 billion annually toward IT procurement. A prevailing industry assumption guarantees that "Mega-Vendors" (top 10 systems integrators) structurally dominate decade-spanning infrastructure deployments due to lobbying and entrenched "vendor lock-in." This project deploys dynamic Random Forest Classifiers against a massive longitudinal dataset of 1,730 high-value IT contracts (FY2018–FY2024) to algorithmically test this assumption.

**Key Mathematical Finding:** Corporate scale wields practically minimal impact on contract longevity past 5 years (<6% Gini Importance). The only statistically significant predictor of multi-generational contract survival is the pure initial financial scope of the primary award (>90% Gini Importance), even within the highly bureaucratized Department of Defense (DoD).

## Repository Architecture

### 1. Data Engineering
*   `data_acquisition.py`: An automated Python pipeline that interfaces dynamically with the **USAspending.gov REST API**. It systematically paginates, implements API rate-limiting, and extracts the highest-funded IT contracts across 4 primary NAICS codes for Fiscal Years 2018 through 2024.

### 2. Exploratory Data Analysis (EDA)
*   `EDA_Contract_Longevity.ipynb`: Jupyter environment mapping the underlying macro trends and distributions. This notebook documents the complete structural failure of generic Prime-to-Subaward linkage reporting within the primary federal endpoints.

### 3. Machine Learning Ecosystem
*   `predictive_modeling.py`: The core Random Forest algorithmic engine. It models the predictive features (Financial Scope vs. Top 10/25/50 Corporate Scale vs. Agency Type) against the critical 5-year federal survival boundary, applying balanced class weights.
*   `temporal_incumbency_tracking.py`: Dynamic recursive modeling that charts the unyielding flatline of incumbency power across four sequential temporal threshold boundaries (1-Year, 3-Year, 5-Year, 10-Year).
*   `plot_agency_importance.py`: Secondary isolation evaluation mapping algorithmic output divergences between the Department of Defense (DoD) versus Civilian bureaucratic agencies.

## Reproducibility Requirements
This repository was engineered for ultimate peer-reviewed transparency. All charts, graphs, ML diagnostics, and temporal dynamics referenced in the scholarly manuscript can be natively generated by running the architecture in sequence.

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Extract Longitudinal Dataset from API (Takes ~3-5 minutes depending on Treasury Servers)
python data_acquisition.py

# 3. Generate Random Forest ML Diagnostics
python predictive_modeling.py

# 4. Map the Temporal Threshold Visuals
python temporal_incumbency_tracking.py
python plot_agency_importance.py
```
